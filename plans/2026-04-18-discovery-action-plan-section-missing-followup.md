# Discovery レポート「アクションプラン」セクション欠損 追加調査プラン

**起票日**: 2026-04-18
**起票元**: `parallel-marinating-nygaard.md`（Phase A/B 検証中に発見）
**優先度**: P1（品質警告が出ている実害ありだが、Phase B コンポーネント実装の問題ではない）
**対象**: `market-lens-ai` バックエンド（プロンプト / LLM 出力 / セクション保証）

---

## 1. 観測された症状

本番 `https://insight-studio-chi.vercel.app/discovery` のカメラの大林レポート（search_id `0cf86273b3f3`）で、画面上部に以下の品質警告が表示されている:

> 品質チェックで注意事項があります
> ・セクション欠損: アクションプランが見つかりません

## 2. 発生条件（コード解析結果）

[backends/market-lens-ai/web/app/report_generator.py:233-241](backends/market-lens-ai/web/app/report_generator.py#L233-L241) の `_quality_gate_check` より、以下の **全条件**が同時に満たされた場合のみ発生:

1. 見出しに「実行プラン」が含まれない
2. かつ、どの見出しにも「実行プラン / アクション / 施策 / 改善提案」のいずれも含まれない

→ **LLM が該当セクションそのものを生成していない**ことを意味する。

## 3. 切り分け済みの事項

- ✅ フロント `PriorityActionHero` は「セクションが欠損していれば null を返す」という正しい仕様で動作（PR #36 Phase B 範囲）
- ✅ PR #36（parallel-marinating-nygaard）の新規改修で**生まれた不具合ではない**（検出の厳格化で顕在化しただけの可能性大）
- ✅ 本プラン（`parallel-marinating-nygaard`）の §9「非ゴール」で「新規レポート生成は対象外」と明記されており、本課題は別プラン対象

## 4. 調査候補

### 4-1. プロンプト調査
- `backends/market-lens-ai/web/app/prompt_builder.py` / プロンプトテンプレートで
  「実行プラン」セクションの出力指示が含まれているか
- Discovery（`discovery_pipeline.py`）側は Compare と別プロンプトのため、欠損している可能性あり

### 4-2. LLM レスポンス監視
- Render logs で直近の Discovery job のレスポンス本文を取得
- 本文中に「実行プラン / 最優先施策 / アクションプラン / 改善提案」見出しが存在するか確認
- トークン切断で尾部が欠落していないか `finish_reason` を確認

### 4-3. セクション保証レイヤ
- もし LLM 出力が不安定なら、`report_generator.py` 側で決定論的に「## 実行プラン」スタブを注入する方針も検討
- 例: 市場規模・競合一覧から「最優先施策 3 件」をテンプレ生成

## 5. 完了判定

- [ ] Discovery レポート 10 件連続で「セクション欠損: アクションプランが見つかりません」が発生しない
- [ ] 既存の Compare レポートにも同症状が出ていないか横展開確認
- [ ] 修正後、`parallel-marinating-nygaard.md` の §3-3 目視検証を再実施して
      `PriorityActionHero` が 3 枚のゴールドカードとして描画されることを確認

## 6. 本プラン起票中に実施済みのハードニング（PR 別途）

PR #XX（本プラン起票と同一コミットチェーン）にて、`PriorityActionHero.jsx` の
検出キーワードをバックエンド品質ゲートと整合させる軽微修正を適用済み:

- 追加対応見出し: `## (広告運用)?アクションプラン` / `## 改善提案`
